- Android 14+ requires `FOREGROUND_SERVICE` and explicit types (e.g. `FOREGROUND_SERVICE_SPECIAL_USE`) when starting Foreground Services. Use `ServiceInfo.FOREGROUND_SERVICE_TYPE_SPECIAL_USE` in `startForeground()`.
- `AlarmManager.setExactAndAllowWhileIdle()` is NOT sufficient for reliable background timing on all devices â€” it can still be suppressed by aggressive battery optimization. Use `setAlarmClock()` for timer/alarm events that must fire at an exact time.
- `setAlarmClock()` requires RTC (wall clock) time, not `ELAPSED_REALTIME_WAKEUP`. When switching, also update elapsed-time calculations from `SystemClock.elapsedRealtime()` to `System.currentTimeMillis()`.
- BroadcastReceivers registered at runtime (`registerReceiver()`) die with the process. For alarms that must survive process death, register the receiver statically in `AndroidManifest.xml`.
- On many OEM Android devices (Oppo/OnePlus/Realme with OplusHansManager, Android 11+), implicit broadcast PendingIntents are blocked with "Background execution not allowed" even when sent via `AlarmManager.setAlarmClock()`. Always use **explicit Intents** (`.setClass(context, ReceiverClass.class)`) in alarm PendingIntents to guarantee delivery.
- **Stale build artifacts cause phantom test failures.** When resource files are deleted or renamed (layouts, strings, IDs), incremental builds can leave outdated R.class/R.txt entries that cause instrumented tests to reference wrong resource IDs. The tests appear to fail with `NoMatchingViewException` even though the code is correct. **Always run `gradle clean` before building after deleting or renaming resource files.** If instrumented tests fail after a resource change, clean build before investigating.
- **Always check GitHub Actions after pushing.** Use `gh run list --limit 3` to verify CI passes. A broken CI pipeline can go unnoticed for many commits if not explicitly checked.
- **Hilt instrumented tests require `HiltTestRunner`** as the test instrumentation runner AND `androidTestAnnotationProcessor 'com.google.dagger:hilt-compiler:...'` in `build.gradle`. Without both, tests fail at runtime with `IllegalStateException` or missing generated component files.
- **Java test class names must exactly match their file names.** A file named `BackupRestoreTest.java` must declare `public class BackupRestoreTest`, not `ZazenTimerActivityTest`. Mismatches cause compilation failures only caught on CI (local `./gradlew build` may not compile androidTest sources).
- **Room schema must exactly match the legacy database schema.** When migrating from raw SQLiteOpenHelper to Room, `PRAGMA table_info()` differences (type case like `integer` vs `INTEGER`, nullable vs NOT NULL) cause `IllegalStateException: Migration didn't properly handle`. Always test migrations on a physical device with real user data â€” the emulator starts with a fresh DB.
- **JUnit `@Rule` ordering is required with Hilt + ActivityScenarioRule.** Without `@Rule(order = 0)` on `HiltAndroidRule` and `@Rule(order = 1)` on `ActivityScenarioRule`, the activity may launch before Hilt injects dependencies. This is device-dependent (fails on API 31, may pass on API 35). `hiltRule.inject()` takes no arguments (not `inject(this)`).
- **Multiple ADB devices cause UTP test runner NullPointerException.** `./gradlew connectedDebugAndroidTest` fails with 0 tests and a NullPointerException when 2 devices are connected. Disconnect one device first, or use `adb shell am instrument` directly on a single device.
- **`./gradlew build` does NOT compile androidTest sources.** Run `./gradlew assembleDebugAndroidTest` separately to catch test compilation errors locally before pushing to CI.
- **`adb logcat --pid=PID` loses tracking when app restarts.** If the app crashes and relaunches, the PID changes and logcat stops capturing. Use tag-based or app-based filtering instead: `adb logcat de.gaffga.android.zazentimer:V *:S`. For full correlation with screen navigation, use unfiltered `adb logcat` piped to a file.
- **`uiautomator dump` can return null root node immediately after app launch.** Wait 2â€“3 seconds after `am start` before dumping. Also, tapping popup menu items via coordinates is fragile â€” always verify the popup items from the dump before tapping.
- **`DbOperations.duplicateSession()` was crashing** with `SQLiteConstraintException: UNIQUE constraint failed: sessions._id`. The method copied the session including its primary key and tried to insert with `@Insert` which conflicted. **Fixed in #52**: set `source.id = 0` before inserting (Room auto-generates). Same pattern already existed for sections. Lesson: when duplicating entities with `@PrimaryKey(autoGenerate = true)`, always reset the PK to 0 before `@Insert`.
- **Espresso `AmbiguousViewMatcherException` with RecyclerView child views.** When a RecyclerView has multiple items, each containing a child view with the same ID (e.g., `sessionOverflow`), `onView(withId(R.id.sessionOverflow))` matches all of them. Use `RecyclerViewActions.actionOnItemAtPosition(pos, clickChildViewWithId(childId))` with a custom `ViewAction` that calls `view.findViewById(id).performClick()` to target a specific item's child view.
- **Gradle `android.injected.signing.store.file` resolves relative to the daemon working directory**, not the project directory. On CI, this means a relative path like `release.keystore` resolves to `~/.gradle/daemon/7.5/release.keystore`. Always use an absolute path (e.g., `$RUNNER_TEMP/release.keystore`).
- **GitHub Actions YAML multi-line `run:` with backslash continuations** can break if indentation is wrong â€” each line may be treated as a separate command. For complex Gradle commands with many `-P` flags, use a single line or a shell script step.
- **LiveData in activity-scoped ViewModels retains stale values across fragment recreations.** When a fragment is destroyed (e.g., via tab navigation) and later recreated, its observer re-subscribes to the same LiveData instance and immediately receives the last emitted value â€” even if that value is now stale (e.g., `running=true` after the service has stopped). Always clear LiveData when the underlying state becomes invalid (e.g., set to null in `stopUpdateThread()`).
- **`showIdleState()` must reset ALL idle-related fields, not just UI visuals.** If a method is called `showIdleState()`, it must set the `meditationRunning` flag to `false` â€” otherwise the UI looks correct but the click handler is routed to the wrong code path.
- **`./gradlew connectedDebugAndroidTest` may fail with protobuf classloader conflicts** on some Gradle versions, even though the tests pass fine via `adb install` + `adb shell am instrument`. If Gradle's UTP runner fails, fall back to manual APK install + `am instrument`.
- **`values/bools.xml` and `values/integers.xml` must NOT be deleted.** They contain AppCompat `abc_*` overrides (`abc_config_closeDialogWhenTouchOutside`, `abc_config_activityDefaultDur`, etc.) that are referenced by the app's theme/styles. Removing them causes `AAPT: error: resource bool/... not found`. They may look like unused library copies, but they are required.
- **`MeditationService.isRunning` must be set to `false` in `onMeditationEnd()`, not just `onDestroy()`.** `stopSelf()` is asynchronous â€” `onDestroy()` runs later. Any code checking `isServiceRunning()` immediately after the end broadcast (e.g., `MainFragment.onResume()`) will see a stale `true` and leave the UI in a disabled state. Setting the flag in `onMeditationEnd()` makes it immediately visible.
- **`unmutePhone()` must not blindly restore ringer state.** Users may manually change the ringer mode during meditation (via hardware keys or settings). Record `mutedRingerMode` when muting, compare to current mode before restoring â€” skip if they differ.
- **`DuplicateSessionTest` fails with `AmbiguousViewMatcherException`** if the database is not reset between runs. Multiple "Copy of" sessions accumulate from previous executions. **Fixed by calling `resetDatabaseForTest()` in `@Before`**.
- **Dim Bell slider is inverted from stored `section.volume`.** SeekBar progress 0 = no dimming (volume=100), progress 90 = max dimming (volume=10). The mapping `volume = 100 - progress` must be maintained in both `fillViewFromData()` and `fillDataFromViews()`. Max progress is 90 (not 100) because 100% dimming is considered useless.
- **Legacy `section.volume` values below 10 must be clamped** in `fillViewFromData()`. Old data may have volume=0, which maps to dimProgress=100, exceeding the SeekBar max of 90. Always clamp to `[0, 90]` and snap to multiples of 10.
- **MainFragment session card tap selects, does NOT navigate.** Tapping a session card only highlights it and saves the selection to preferences. To reach `SessionEditFragment`, tap the three-dot overflow menu on the card and choose "Edit Session" â€” or create a new session via FAB which auto-navigates to edit.
- **`uiautomator dump` default path not writable on API 29 emulator.** The default path `/sdcard/window_dump.xml` is not writable. Use an explicit path: `adb shell uiautomator dump /data/local/tmp/window_dump.xml` then `adb pull`.
- **Emulator API 29 cold boot shows ANR dialog.** "System UI isn't responding" dialog appears during cold boot and blocks `input tap` commands. Dismiss it by tapping "Wait" before interacting.
- **Emulator StorageManager race after boot.** `adb install` fails with `NullPointerException: StorageManager.getVolumes() returns null` shortly after `sys.boot_completed=1`. Wait until `adb shell pm list packages` returns results (~10s after boot_completed) or kill and restart the emulator.
- **`pm` service not ready at `sys.boot_completed=1`.** On API 29 emulator, `adb shell pm install` returns "Can't find service: package" for ~40s after cold boot even though boot_completed=1. Wait for `pm list packages` to succeed before installing.
- **Emulator reboot needs ~45s for stable state.** After `adb reboot`, `sys.boot_completed=1` at ~30s but system services remain unstable. Add 15s additional wait before interacting.
- **Batch translations via deep_translator can time out.** 12 languages Ă— 141 strings = ~1700 API calls. With `sleep(0.05)` per call, this takes ~7-10 minutes. Use a large timeout (â‰Ą20 minutes) and consider splitting runs or reducing sleep to 0.01s. Non-Latin scripts (Odia, Pashto) are especially vulnerable to format specifier corruption â€” always verify with `grep -c 'â¦…'` after translation to confirm no leftover placeholders.
- **googletrans vs deep-translator.** `googletrans==4.0.0-rc1` returns empty JSON for many languages â€” don't use it. Use `deep-translator` (GoogleTranslator) instead: `pip install deep-translator`.
- **Format specifier corruption by Google Translate.** Non-Latin scripts (Amharic, Georgian, Burmese, Khmer, Thai, Arabic, Hindi) corrupt `%1$d` and `%1$s` by converting Latin characters to local-script equivalents (`%1$d` â†’ `%1$Đ´` in Cyrillic). Mask ALL format specifiers with ASCII placeholder tokens (`__FMT0__`, `__NL__`) before sending to Google Translate. Unicode bracket placeholders (`â¦…0â¦†`) are themselves corrupted in some scripts.
- **`\n` corruption in segment translation.** When translating `help_sectionlist_text`, split by `\n`, translate each segment individually, then rejoin. Batch/single-pass translation corrupts `\n` sequences in RTL and non-Latin scripts.
- **`character_counter_pattern` must never be translated.** `%1$d / %2$d` is a pure format pattern â€” Google Translate converts digits to local numerals even in Latin-script languages.
- **AAPT2 rejects raw apostrophes in string values.** Replace `'` â†’ `\'` in output XML. Use regex `(?<!\\\\)'` to skip already-escaped `\'`.
- **Short strings may return untranslated.** Google Translate sometimes returns the source unchanged for very short strings (e.g., "OK", "Help", section names). This is acceptable.
